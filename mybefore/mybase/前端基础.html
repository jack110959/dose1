# （5）定位

# 相对定位

相对自身原本位置进行移动

设置之后将该元素独立出来，修改位置后不再与其他元素相互影响。

position: relative;
# 绝对定位

相对设置了相对定位的父级元素进行移动

 position: absolute;

需要注意：这种用法适用于父级元素不会移动的情况下，不然子元素也会随着发生改变

设置绝对定位之后，设置top等属性的参照为body

但如果父元素设置了相对定位，那么参照点将为设置了相对定位的父级元素

 <div class="par">
       <div class="child"></div>
</div>

<style>
    .par{
        width: 800px;
        height: 100px;
        border: solid 2px orange;
        /* 下面如果没有设置，参照就为body */
        position: relative;
    }
    .child{
        width: 40px;
        height: 40px;
        border: solid 2px blue;
        position: absolute;
        top: 100px;
    }
</style>
# 固定定位

随着页面上下滚动，始终保持在同一个位置。

position: fixed;
# 调节属性
1.top等为绝对定位，需与position：absolute一起用才有效；而margin-top为相对定位；

2.绝对定位一般情况下以body为标准；若父元素设置position：relative，，则以其父元素为标准

# （3）运算符

# 等值运算符==

先判断类型，类型一致则直接比较

​ 类型不一致，则先使用Number进行强转后再进行比较

var a=1;
var a1="1";
var a2=true;
var a3="true";
var a4="a";

alert(a==a2);//true
alert(a==a3);//false
alert(a1==a2);//true
alert(a1==a3);//false
alert(a2==a3);//false

# 等同运算符===

先判断类型，类型一致则再判断值，值相同则返回true。

​ 类型不一致则直接返回false。

# 特殊情况

//true

//类型不一致，强转后如下
alert(Number(null))//0

//false  类型不一致，null为object类型，undefined依旧为undefined类型
alert(null===undefined);

# window对象的方法

alert： 警告框 提示一个警告信息，没有返回值

confirm：确认框 提示用户选择一项操作（确定/取消）

​ 点击确定 返回true

​ 点击取消 返回false

prompt： 提示框 提示用某个信息的录入或者说收集

​ 点击确定，返回当前用户输入的数据，默认返回空字符串

​ 点击取消，返回null

    }
    function test2(){
        var the_return=window.confirm("你确定要删除吗");

    }
    function test3(){
        var the_return=window.prompt("请输入你的密码");
        alert(the_return);
    }

setTimeout：定时执行，设置几秒后执行一次该函数

参数一，一个执行函数

参数二，时间，单位毫秒

返回值：当前定时器的id

function test4(){
    var the_return=window.setTimeout(function(){alert("定时2秒后执行")},2000);

}

参数一，一个执行函数

参数二，时间，单位毫秒

返回值：当前间隔器的id

function test5(){
    var the_return=window.setInterval(() => {
        alert("间隔执行")
    }, 3000);
}

# （11）表单输入框的校验

# 普通思路

采用正则表达式校验，这里只举例用户名校验，其他同理。

<form action="">
        用户名：<input type="text" id="name" onblur="checkname()"><span id="namespan" >输入2到4为汉字</span>
</form>

js代码

function checkname(){
        var name=document.getElementById("name").value;
        var span=document.getElementById("namespan");
       //符合两个到4个汉字汉字
        var reg=/^[\u4e00-\u9fa5]{2,4}$/;

        if(name==""|| name==null){
            span.innerHTML="用户名不能为空";
            span.style.color="red";
        }else if(reg.test(name)){
            span.innerHTML="用户名ok";
            span.style.color="green";
        }else {
            span.innerHTML="用户名不合法";
            span.style.color="red";
        }
    }
# 封装

经过上面的练习，你会发现大部分的表单验证逻辑基本一致，所以这时候就必须要想到采用封装来实现这部分功能。

思路：定义一个方法来处理所有的输入框验证。

html代码

<!--需要编写alt属性值，用来输出提示信息-->
<form action="">
用户名：<input type="text" id="name" alt="用户名" 
onblur="check(this.id,/^[\u4e00-\u9fa5]{2,4}$/)"><span id="nameSpan" >输入2到4为汉字</span>
</form>

js代码

//封装方法，两个形参，id用来获取元素，reg用来做正则表达式校验
    function check(id,reg){
        var name=document.getElementById(id).value;
        var alt=document.getElementById(id).alt;
        //这里需要根据span的id命名规则进行相应改动
        var span=document.getElementById(id+"Span");
        if(name==""|| name==null){
            span.innerHTML=alt+"不能为空";
            span.style.color="red";
        }else if(reg.test(name)){
            span.innerHTML=alt+"ok";
            span.style.color="green";
        }else{
            alert(alt+"不合法");
            span.innerHTML=alt+"不合法";
            span.style.color="red";
        }
    }

# form表单的方法执行顺序

对每个校验方法都设置返回值，然后提交再统一执行一遍所有方法，然后返回所有校验方法返回值的与值。

onsubmit只能表单上使用,提交表单前会触发, onclick是按钮等控件使用, 用来触发点击事件。

onsubmit处理函数返回false，onclick函数返回false，都不会引起表单提交。

<form action="${pageContext.request.contextPath }/login.action" 
			                       method="post" onsubmit="return check()">
<script>
// 判断是登录账号和密码是否为空
function check(){
    var usercode = $("#usercode").val();
    var password = $("#password").val();
    if(usercode=="" || password==""){
    	$("#message").text("账号或密码不能为空！");
        return false;
    }  
    return true;
}
</script>

在提交表单前，一般都会进行数据验证，可以选择在submit按钮上的onclick中验证,也可以在onsubmit中验证。

但是onclick比onsubmit更早的被触发。

提交过程

1、用户点击按钮 —->

2、触发onclick事件 —->

3、onclick返回true或未处理onclick —->

4、触发onsubmit事件 —->

5、onsubmit未处理或返回true ——>

6、提交表单.
