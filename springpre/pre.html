# Spring前言

# 一代理模式

# 1，静态代理

代理类在程序运行前就已经定义好，其与目标类的关系在程序运行前就已经确立，代理类与目标类实现同一个接口，在代理类中创建目标对象，再调用对应的方法，获取方法返回值后再进行增强处理，代理类代码如下

// 代理类
public class SomeServiceProxy implements ISomeService {

	private ISomeService target;
//使用构造器在实例化时对target进行初始化
	public SomeServiceProxy() {
		target = new SomeServiceImpl();
	}

	@Override
	public String doFirst() {
		// 代理类调用目标方法
		String result = target.doFirst();
		// 增强就发生在这里
		return result.toUpperCase();
	}

	@Override
	public void doSecond() {
		target.doSecond();
	}

}

# 2，动态jdk代理

动态代理在整个程序运行时不存在目标类的代理类，目标类的代理对象只是由代理生成工具（如代理工厂类）在程序运行时由jvm根据反射等机制动态生成。代理对象与目标对象的代理关系在程序运行时才确立。

每次调用方法时都会在invoke方法内进行增强，result为方法返回值。

public class MyTest {

	public static void main(String[] args) {
        //匿名内部类的传入值类型必须为final
		final ISomeService target = new SomeServiceImpl();
		
		// 使用JDK的Proxy动态代理，要求目标类必须实现接口
		// 因为其底层的执行原理，与静态代理的相同
		ISomeService service = (ISomeService) Proxy.newProxyInstance(
					target.getClass().getClassLoader(),    // 目标类的类加载器
					target.getClass().getInterfaces(),     // 目标类所实现的所有接口
					new InvocationHandler() {              // 匿名内部类
						
						// proxy：代理对象
						// method：目标方法
						// args：目标方法的参数列表
						@Override
						public Object invoke(Object proxy, Method method, Object[] args)
								throws Throwable {
							// 调用目标方法
							Object result = method.invoke(target, args);
							if (result != null) {
								result = ((String) result).toUpperCase();
							}
							return result;
						}
					});
		String result = service.doFirst();
		System.out.println("result = " + result);
		service.doSecond(); 
	}

}

# 3，CGLIB动态代理

适用于无接口的类，原理为通过继承的方式生成目标类的子类，在子类中进行增强处理，所以子类对象也就是代理对象，需要注意的由于是通过继承的方式，所以要求目标类不能是final类型的类。由于接口也可以拥有子类，所有CGLIB动态代理模式也适用于有接口的目标类。

下面为代理工厂类，方法回调设计模式主要体现在，假定MyCglibFactory为A类，enhancer为B类，A类调用了B类setCallback方法并将回调对象this作为参数传递给了B类，B类对象又调用了A类对象的intercept方法，所有intercept方法为回调方法

public class MyCglibFactory implements MethodInterceptor {
	
	private SomeService target;
//通过修改构造器即可生成自己需要的代理对象
	public MyCglibFactory() {
		target = new SomeService();
	}
//在测试类中使用new MyCglibFactory().myCglibCreator()及可获取代理对象
	public SomeService myCglibCreator() {
		// 创建增强器对象
		Enhancer enhancer = new Enhancer();
		// 指定目标类，即父类
		enhancer.setSuperclass(SomeService.class);
		// 设置Callback回调接口对象
		enhancer.setCallback(this);
		
		return (SomeService) enhancer.create();
	}

	// 回调方法，在intercept回调方法中进行增强处理
	@Override
	public Object intercept(Object obj, Method method, Object[] args,
			MethodProxy proxy) throws Throwable {
		// 调用目标方法
		Object result = method.invoke(target, args);
		if (result != null) {
			result = ((String) result).toUpperCase();
		}
		return result;
	}

}

# 二适配器模式

适配器的定义是将某一个类的接口转换为客户所需的接口类型，换句话说，适配器模式解决的问题是，使得原本由于接口不兼容而不能一起工作的，不能统一管理的那些类可以一起工作，可以进行统一管理。

基本原理构造一个适配器接口，该接口具有一个带object类型形参的方法，再编写适配器接口实现类，在实现类方法中将传入的对象强转成对应的对象，再进行具体的操作。

（1）下面项目为多个工种类共用一个适配器并且

省略了ICooker和IProgrammer接口及其对应的实现类，以及适配器接口

适配器接口实现类代码如下

// 适配器类
//使用instanceof判断传入的是哪个类型的对象再进行强转
public class WorkerAdapter implements IWorkerAdapter {

	@Override
	public String work(Object worker) {
		String workContent = "";
		// 若传来的对象是厨师，则调用其cook()方法
		if(worker instanceof ICooker) {
			workContent = ((ICooker)worker).cook();
		} else if(worker instanceof IProgrammer) {
			workContent = ((IProgrammer)worker).program();
		}
		
		return workContent;
	}

}

测试类代码如下

public class MyTest {

	public static void main(String[] args) {
		ICooker qjdCooker = new QjdCooker();
		IProgrammer jdProgrammer = new JdProgrammer();
		Object[] workers = {qjdCooker, jdProgrammer};
		
		// 创建适配器对象
		IWorkerAdapter adapter = new WorkerAdapter();
		// 循环遍历每个工种对象，让每个工种对象在适配器中逐个进行匹配
		for (Object worker : workers) {
			String workContent = adapter.work(worker);
			System.out.println(workContent);
		}
	}

}

（2）下面项目为每一个工种分配一个适配器

在这里依旧省略了ICooker和IProgrammer接口及其对应的实现类。

适配器接口代码如下，第一个方法用来进行类型转换，第二个方法用来进行对象判断。

public interface IWorkerAdapter {
	String work(Object worker);
	boolean supports(Object worker);
}

然后再为每一个工种创建一个适配器，所有适配器都实现了上面的接口，这里只列举一个实现类。

public class CookerAdapter implements IWorkerAdapter {

	@Override
	public String work(Object worker) {
		return ((ICooker)worker).cook();
	}

	@Override
	public boolean supports(Object worker) {
		return (worker instanceof ICooker);
	}

}

最后在测试类中进行使用

public class MyTest {

	public static void main(String[] args) {
		ICooker qjdCooker = new QjdCooker();
		IProgrammer jdProgrammer = new JdProgrammer();
		Object[] workers = {qjdCooker, jdProgrammer};
		
		// 循环遍历每个工种对象，让每个工种对象在适配器中逐个进行匹配
		for (Object worker : workers) {
			IWorkerAdapter adapter = getAdapter(worker);
			System.out.println(adapter.work(worker));
		}
	}

	// 根据worker获取相应的适配器对象，调用每一个适配器的supprts方法来判断需要返回哪个适配器
	private static IWorkerAdapter getAdapter(Object worker) {
		List<IWorkerAdapter> adapters = getAllAdapters();
		for (IWorkerAdapter adapter : adapters) {
			if(adapter.supports(worker)) {
				return adapter;
			}
		}
		return null;
	}

	// 获取所有的适配器，将所有的适配器实现类实例化后存入到list中，最后返回list
	private static List<IWorkerAdapter> getAllAdapters() {
		List<IWorkerAdapter> adapters = new ArrayList<>();
		adapters.add(new CookerAdapter());
		adapters.add(new ProgrammerAdapter());
		return adapters;
	}

}

# 三模板方法设计模式

定义：定义一个操作某种算法的框架，而将一些步骤延迟到子类中，模板方法模式使得子类在不改变一个算法结构的前提下，对某些步骤实现个性化定义。

在模板方法设计模式中，存在一个父类。其中包含两类方法：模板方法和步骤方法。

模板方法：

即实现某种算法的方法步骤。而这些步骤都是调用步骤方法完成的。

步骤方法：

即完成模板方法的每个阶段性方法。

步骤方法有三种类型：抽象方法，最终方法和钩子方法。

抽象方法：是要求子类必须实现的方法，是完成模板方法的算法步骤中必须由子类完成的个性化定义。

最终·方法：是子类不能重写的方法，是若要完成模板方法的算法步骤，对于所有的子类执行一样的步骤。

钩子方法：是父类给出了默认实现，但子类也可以重写的方法。

例子：

先定义一个抽象类作为父类

public abstract class Shopping {
	
	// 模板方法
	public void buyGoods() {
		userLogin();
		buy();
		pay();
	}

	// 子类不能重写的方法
	public final void userLogin() {
		System.out.println("用户登录");
	}
	
	// 子类必须实现
	public abstract void buy();
	
	// 钩子方法，子类可重写的方法
	public void pay() {
		System.out.println("使用银联卡支付");
	}
}

定义两个子类

//子类ShoesShopping
public class ShoesShopping extends Shopping {

	@Override
	public void buy() {
		System.out.println("购买红蜻蜓皮鞋");
	}
}

//子类ClothesShopping，两个子类的实现了抽象方法buy（）,该子类也重写了钩子方法pay（）
public class ClothesShopping extends Shopping {

	@Override
	public void buy() {
		System.out.println("购买七匹狼男装");
	}
	
	@Override
	public void pay() {
		System.out.println("使用支付宝付款");
	}

}

定义测试类

public class MyTest {
	public static void main(String[] args) {
		Shopping shoesShopping = new ShoesShopping();
		shoesShopping.buyGoods();
		System.out.println("-----------------------");
		Shopping clothesShopping = new ClothesShopping();
		clothesShopping.buyGoods();
	}
}
